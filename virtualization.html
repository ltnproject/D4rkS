Oracle VirtualBox

Recommended Version: 7.0+

TypeLinux / Ubuntu (64-bit)
Memory4096 MB (Min)
Processors2 Cores (Min)
GraphicsVMSVGA (128MB VRAM)
  • 1

    Enable EFI (Critical)

    Go to Settings > System > Motherboard and check Enable EFI (special OSes only). D4rkzOS uses systemd-boot which requires UEFI.

  • 2

    Acceleration Settings

    Under System > Processor, ensure PAE/NX is enabled. Under Display, check Enable 3D Acceleration for smooth KDE Plasma animations.

  • 3

    Mount the ISO

    Go to Storage, click the empty optical drive, and select your D4rkzOS-2.0.iso.

Known Issue: If you see a black screen on boot, ensure Graphics Controller is set to VMSVGA and not VBoxSVGA.
⚡

VMware Workstation / Player

Recommended: v17.x

OS SelectionUbuntu 64-bit
Firmware TypeUEFI
Memory4 GB+
GraphicsAccelerated 3D
  • 1

    Firmware Calibration

    During VM creation, ensuring UEFI is selected as the firmware type. If you've already created it, check VM Settings > Options > Advanced > Firmware type.

  • 2

    IO Performance

    Set the Virtual Disk type to NVMe or SATA (avoid IDE) for the best simulated throughput.

  • 3

    Post-Install

    Run sudo apt install open-vm-tools-desktop after installation for clipboard sharing and resolution resizing.

📱

UTM (Apple Silicon / Intel)

Modern virtualization for macOS

ModeVirtualize (Not Emulate)
Architecturex86_64
SystemQ35 + UEFI
Displayvirtio-ramfb-gl (GPU)
  • 1

    Virtualize vs Emulate

    If you are on an Intel Mac, use Virtualize. If you are on Apple Silicon (M1/M2/M3), you must use Emulate (which will be slower) unless you have an ARM64 build of D4rkzOS.

  • 2

    Display Optimization

    For smooth KDE Plasma performance, ensure virtio-ramfb-gl is selected and **Hardware OpenGL** is enabled in UTM settings.

  • 3

    Boot Selection

    Once the ISO is attached, ensure it is the first item in the boot order or use the boot menu during the UEFI logo.
